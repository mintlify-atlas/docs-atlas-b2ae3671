RetainPDF translates PDF documents — including scanned and image-based files — while preserving the original layout, inline math formulas, tables, and typographic structure. It delivers translated output as PDF, Markdown, or a ZIP bundle, and can be accessed through a web browser, a desktop application, or the REST API.

## How it works

RetainPDF runs a four-stage pipeline for every document:

<Steps>
  <Step title="Upload">
    Upload a PDF through the web UI, desktop app, or `POST /api/v1/uploads`. RetainPDF accepts both text-based and image/scanned PDFs.
  </Step>
  <Step title="OCR and normalize">
    The document is processed by your chosen OCR provider (MinerU or PaddleOCR), then normalized into a structured internal format that preserves geometry, layout roles, and formula positions.
  </Step>
  <Step title="Translate">
    Each text block is translated using your configured LLM (such as DeepSeek). Translation rules, glossaries, and skip policies let you control exactly what gets translated.
  </Step>
  <Step title="Render and download">
    Translated content is rendered back into PDF using Typst, preserving the original page layout and font sizing. Download the result as PDF, Markdown, or a ZIP bundle.
  </Step>
</Steps>
